## Purpose and Scope

This document covers SillyTavern's text completion system, which supports 15+ self-hosted and commercial backends using the text completion paradigm (prompt → text continuation). This system manages the lifecycle of text generation from configuration and model discovery to payload transformation and streaming response handling.

Text completion backends are characterized by:

- Single prompt string input (managed by the Prompt Assembly Pipeline).

- Direct text continuation output.

- Extensive sampling parameter support (Temperature, Top-P, Min-P, etc.).

- Specialized local inference features (DRY, XTC, Mirostat).

For message-based APIs (OpenAI, Claude, etc.), see the Chat Completion System.

## Backend Architecture

SillyTavern utilizes a proxy architecture where the client communicates with a unified server-side router, which then dispatches requests to specific backend adapters.

## Configuration and Sampler Management

### Sampler Ordering Logic

Different inference engines require specific sampler execution sequences. SillyTavern maintains default orders for major backends.

- llama.cpp: Defined by LLAMACPP_DEFAULT_ORDER including penalties, dry, top_n_sigma, top_k, typ_p, top_p, min_p, xtc, temperature, and adaptive_p. public/scripts/textgen-settings.js67-78 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/textgen-settings.js#L67-L78)

- Ooba (text-generation-webui): Defined by OOBA_DEFAULT_ORDER, prioritizing repetition_penalty, dry, and temperature. public/scripts/textgen-settings.js79-101 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/textgen-settings.js#L79-L101)

- Aphrodite: Defined by APHRODITE_DEFAULT_ORDER, prioritizing dry, penalties, and no_repeat_ngram. public/scripts/textgen-settings.js102-117 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/textgen-settings.js#L102-L117)

- KoboldCpp: Uses a numeric index array KOBOLDCPP_ORDER ([6, 0, 1, 3, 4, 2, 5]). public/scripts/textgen-settings.js143 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/textgen-settings.js#L143-L143) public/scripts/kai-settings.js76 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/kai-settings.js#L76-L76)

## Backend-Specific Implementations

### Ollama Integration

Ollama requires specialized handling due to its unique API structure.

- Model Discovery: Uses /api/tags instead of the standard /v1/models. src/endpoints/backends/text-completions.js146-147 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/text-completions.js#L146-L147)

- Streaming: Implements parseOllamaStream to manually parse non-standard JSON chunks into valid Server-Sent Events (SSE). src/endpoints/backends/text-completions.js29-72 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/text-completions.js#L29-L72)

- Multimodal: Supports image captioning via the /api/generate endpoint with an images array. src/endpoints/backends/text-completions.js476-518 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/text-completions.js#L476-L518)

### KoboldCpp and AI Horde

- Status: Checked via native /v1/info/version or /extra/version. src/endpoints/backends/kobold.js163-172 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/kobold.js#L163-L172)

- Abortion: Implements abortKoboldCppRequest (server-side) and local listeners in kobold.js which send a POST to /extra/abort when a generation is cancelled by the user. src/endpoints/backends/text-completions.js80-97 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/text-completions.js#L80-L97) src/endpoints/backends/kobold.js21-38 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/kobold.js#L21-L38)

## Tokenization and Token Management

### Local and Remote Tokenizers

SillyTavern supports both local (client-side and server-side) and remote (API-side) tokenization.

- Local Tokenizers: Includes GPT2, OpenAI (Tiktoken), Llama (SentencePiece), Mistral, and others. public/scripts/tokenizers.js16-38 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/tokenizers.js#L16-L38)

- Server-side Execution: The src/endpoints/tokenizers.js file handles intensive tokenization tasks using @agnai/web-tokenizers and tiktoken. src/endpoints/tokenizers.js11-13 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/tokenizers.js#L11-L13)

- Remote Tokenization: Probes for support via backend-specific endpoints (e.g., Ooba internal info). src/endpoints/backends/text-completions.js190-201 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/backends/text-completions.js#L190-L201)

### Tokenizer Caching

To improve performance, token counts are cached in an objectStore (via localforage) under the name SillyTavern_ChatCompletions. public/scripts/tokenizers.js157-159 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/tokenizers.js#L157-L159)

- Loading: loadTokenCache() retrieves cached counts on initialization. public/scripts/tokenizers.js171-179 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/tokenizers.js#L171-L179)

- Saving: saveTokenCache() persists the tokenCache object. public/scripts/tokenizers.js181-189 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/tokenizers.js#L181-L189)

## Authors Note and Floating Prompts

The system supports "floating" prompt injections through the Authors Note system.

- Metadata: Uses keys like note_prompt, note_depth, and note_role to store settings per chat. public/scripts/authors-note.js30-36 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/authors-note.js#L30-L36)

- Injection Depth: Managed by onExtensionFloatingDepthInput, which ensures the depth is a positive integer. public/scripts/authors-note.js149-160 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/authors-note.js#L149-L160)

- Token Counting: Dynamically updates token counters for main, character, and default notes using getTokenCountAsync. public/scripts/authors-note.js132-134 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/authors-note.js#L132-L134)
