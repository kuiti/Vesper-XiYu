### Description

### Description

When using the newly released deepseek-v4-pro model via the openai-compat provider, any tool call triggered by the model results in a 400 Bad Request from the DeepSeek API.
Error Message: ERROR Bad Request: The reasoning_content in the thinking mode must be passed back to the API

### Root Cause Analysis

DeepSeek's API has a strict context requirement for reasoning models: if a model makes a Tool Call, the client MUST pass the reasoning_content from that turn back to the API in the subsequent request.
