## Purpose and Scope

The Chat Completion System handles message-based interactions with Large Language Models (LLMs) using the ChatML (Chat Markup Language) message format. This system provides a unified interface for communicating with 20+ chat completion API providers including OpenAI, Claude, Google AI Studio, Mistral, Groq, and many others.

Related pages:

- For text completion APIs (KoboldAI, Oobabooga, etc.), see 3.2 Text Generation System (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/3.2 Text Generation System)

- For provider-specific prompt format conversion, see 3.3 Prompt Assembly Pipeline (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/3.3 Prompt Assembly Pipeline)

- For tool calling capabilities, see 3.5 Tool Calling and Function Execution (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/3.5 Tool Calling and Function Execution)

- For reasoning/thinking content display, see 3.8 Reasoning Display System (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/3.8 Reasoning Display System)

The system manages:

- Message formatting: Converting internal chat history to provider-specific formats.

- Authentication: Secure API key management via the Secrets system.

- Request routing: Backend proxy layer for multiple providers.

- Response handling: Both streaming (SSE) and non-streaming responses.

- Multimodal content: Image, video, and audio attachments in messages.
