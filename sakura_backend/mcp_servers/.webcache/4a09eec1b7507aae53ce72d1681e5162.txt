This document covers SillyTavern's vector storage and Retrieval Augmented Generation (RAG) system, which enables semantic search and context retrieval through text embeddings. The system vectorizes chat messages, file attachments, and World Info entries, storing them in local vector databases for similarity-based retrieval during chat generation.

For information about World Info keyword-based activation, see 6.1 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/6.1) For chat summarization features, see 6.2 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/6.2)

## Purpose and Scope

The Vector Storage system provides three primary capabilities:

- Chat Message RAG: Retrieves semantically relevant past messages and injects them into prompts public/scripts/extensions/vectors/index.js85-93 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L85-L93)

- File Vectorization: Chunks and indexes large text files for semantic search public/scripts/extensions/vectors/index.js97-103 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L97-L103)

- Vector-Based World Info: Activates World Info entries based on semantic similarity rather than keywords public/scripts/extensions/vectors/index.js117-120 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L117-L120)

The system supports 20+ embedding providers src/endpoints/vectors.js22-43 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/vectors.js#L22-L43) and uses the vectra library for local vector storage with cosine similarity search src/endpoints/vectors.js4 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/vectors.js#L4-L4)

## Vector Storage Architecture

### Vectra Integration

SillyTavern uses the vectra package to manage local vector indices src/endpoints/vectors.js4 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/vectors.js#L4-L4) Indices are partitioned by user, source, collection, and model src/endpoints/vectors.js273-283 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/vectors.js#L273-L283)

- Collection ID: Usually the chatId for chat logs public/scripts/extensions/vectors/index.js338 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L338-L338) or a hash of the file URL for attachments public/scripts/extensions/vectors/index.js128-130 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L128-L130)

- Storage Path: DATA_ROOT/vectors/<source>/<collectionId>/<model>/ src/endpoints/vectors.js273-283 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/vectors.js#L273-L283)

### Data Schema

Items stored in the index contain the vector and metadata src/endpoints/vectors.js303-311 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/vectors.js#L303-L311):

- vector: The embedding array.

- metadata: Object containing the original text, a unique hash, and the index (position).

### Synchronization

## World Info Vector Activation

- Query Generation: A query is built from recent chat history public/scripts/extensions/vectors/index.js1296 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L1296-L1296)

- Similarity Search: World Info entries are compared against the query vector public/scripts/extensions/vectors/index.js1344-1358 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/extensions/vectors/index.js#L1344-L1358)
