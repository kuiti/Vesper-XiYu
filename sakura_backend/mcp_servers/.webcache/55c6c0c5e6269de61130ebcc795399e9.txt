## Purpose and Scope

This document covers the management of character cards in SillyTavern: their storage format, import/export mechanisms, caching strategies, and API endpoints for character operations. Character cards contain the personality, description, and metadata for AI characters that users interact with.

For information about chat message storage, see Chat Storage and Persistence. For character sprite assets and expressions, see Asset and File Management.

## Character Card Specification

### Spec V3 Format (Current)

SillyTavern supports the Character Card V3 specification. Data is embedded as JSON metadata within PNG image files using tEXt chunks. V3 takes precedence over V2 during the read process.

### Spec V2 and V1 (Legacy)

SillyTavern maintains backwards compatibility with V2 (chara keyword in PNG) and V1 (flat JSON). When writing a card, SillyTavern generates both V2 and V3 chunks to ensure the card remains readable by older software while supporting new features.

## Character Card Reading and Caching

### Caching Architecture

SillyTavern employs a multi-tiered caching system to handle large character libraries efficiently, especially on mobile devices.

- MemoryLimitedMap: An in-memory cache with a configurable size (default 100MB). It evicts old entries when the limit is reached.

- DiskCache: A class that uses node-persist to store parsed JSON results on disk, avoiding expensive PNG metadata extraction on subsequent loads. It includes a SYNC_INTERVAL (5 minutes) for cache maintenance.

Sources: src/endpoints/characters.js31-38 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/characters.js#L31-L38) src/endpoints/characters.js38-157 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/characters.js#L38-L157) src/endpoints/characters.js162-209 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/src/endpoints/characters.js#L162-L209)

### Lazy Loading (Shallow Characters)

When performance.lazyLoadCharacters is enabled in config.yaml, the server returns "shallow" character objects to the client. These contain only essential metadata (name, avatar, tags) to populate the character list without loading full descriptions or lorebooks into memory.

### BYAF Parsing Details

The ByafParser class handles the extraction of "Backyard Archive Format" files. It maps Backyard-specific macros (e.g., #{character}:) to SillyTavern macros (e.g., {{char}}:).

- Lorebooks: Converts loreItems into SillyTavern CharacterBook entries via convertCharacterBook().

- Greetings: Maps scenarios to alternate_greetings using formatAlternateGreetings().

- Images: Extracts character icons from the ZIP buffer using getCharacterImages().

## Image Processing and Writing

### applyAvatarCropResize

SillyTavern standardizes character avatars to 400x600 pixels using the Jimp library.

- Cropping: If the user provides crop coordinates (x, y, width, height), the image is cropped.

- Resizing: The image is resized to AVATAR_WIDTH and AVATAR_HEIGHT.

- Encoding: The result is encoded as a PNG.

- Metadata Injection: The character JSON is base64 encoded and injected into the tEXt chunks via the write function in character-card-parser.js.

## Recent Chats and Pinned State

The welcome-screen.js module manages the display of recent and pinned character interactions on the dashboard.

### PinnedChatsManager

This class handles persistence of pinned status using accountStorage.

- toggle(recentChat, pinned): Toggles the pinned state.

- rename(recentChat, newFileName): Migrates pinned status when a chat file is renamed.

- isPinned(recentChat): Checks if a chat is in the pinnedChats storage.

### Character Export

Characters are exported via POST /api/characters/export. The server:

- Reads the character data.

- Calls unsetPrivateFields() to remove local-only data like fav (favorite status) and chat (current chat filename).

- Sends the PNG or JSON file to the client.
