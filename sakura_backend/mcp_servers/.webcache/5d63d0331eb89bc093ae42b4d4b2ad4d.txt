This page provides an overview of SillyTavern's context and memory management systems. These systems augment AI prompts with additional information to improve response quality and consistency. Three main subsystems work together to provide different types of context:

- World Info/Lorebooks: Keyword-triggered lore injection (detailed in World Info System).

- Chat Summarization: Periodic conversation summaries (detailed in Chat Summarization Extension).

- Vector Storage/RAG: Semantic retrieval of relevant information (detailed in Vector Storage and RAG System).

This page focuses on how these systems integrate into the prompt assembly pipeline and interact with each other.

## Purpose in the Generation Pipeline

Context and memory systems solve a fundamental problem: AI models have limited context windows, but conversations and character lore can be extensive. These systems intelligently select and inject the most relevant information into each generation request via the PromptManager and extension hooks.

## System Architecture Overview

Each context system operates independently but coordinates through the extension prompt mechanism defined in extensions.js. They run at different stages of the generation process and inject content at configurable positions like IN_PROMPT, BEFORE_PROMPT, or AT_DEPTH.

## World Info System Overview

The World Info system (also called "Lorebooks") provides keyword-activated lore injection. When specific keywords appear in the chat, corresponding entries are inserted into the prompt based on the world_info_insertion_strategy public/scripts/world-info.js27-31 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/world-info.js#L27-L31)

### Key Components

- Entries: Individual lore pieces with activation keys, content, and metadata.

- Scanning: The system evaluates chat history based on world_info_depth public/scripts/world-info.js69 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/world-info.js#L69-L69)

- Activation Logic: Supports complex triggers including AND_ANY, NOT_ALL, NOT_ANY, and AND_ALL public/scripts/world-info.js33-38 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/world-info.js#L33-L38)

## Chat Summarization Extension

The Memory extension periodically generates summaries of the conversation to preserve important context while staying within token limits. It utilizes extension_prompt_roles and extension_prompt_types to inject the summary into the prompt assembly pipeline.

### Key Components

- Trigger Mechanism: Periodic updates based on prompt intervals or manual force.

- Prompt Builders: Supports multiple strategies for formatting the summary block within the context.

- Injection: Adds summary to prompt via setExtensionPrompt using the identifier 1_memory.

Detailed configuration and behavior is documented in Chat Summarization Extension.

## Vector Storage and RAG System

The Vectors extension provides semantic retrieval of relevant information from long-term chat history or external files (Data Bank). It interfaces with the server via the /api/vectors endpoints.

### Key Components

- Vectorization: Converts text to embeddings using various providers (OpenAI, Transformers, etc.).

- Storage: Uses vector indices on the server to manage local storage of embeddings.

- Retrieval: Performs similarity searches against the current chat context to find the most relevant "memories" and injects them into the prompt.

## Integration with Prompt Assembly

All context systems inject content through the extension prompt mechanism. This provides a unified interface for adding context at specific positions in the prompt, managed by the PromptManager.
