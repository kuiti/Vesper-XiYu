## Purpose and Scope

The prompt assembly pipeline is a multi-stage process that constructs the final context sent to Large Language Models (LLMs). It handles the aggregation of disparate data sources—character definitions, user personas, chat history, World Info (Lorebooks), and extension data—and formats them into a single payload. This system is governed by the PromptManager class and influenced by Instruct Mode and Context Templates. Depending on the target API, the output is either a structured message array (Chat Completion) or a raw string (Text Completion).

## Pipeline Overview

The pipeline transforms raw data into a structured format suitable for the target API. SillyTavern branches this logic based on the main_api selected by the user.

### Prompt Injection Logic

Prompts can be injected at different positions relative to the chat history. This is defined by the INJECTION_POSITION enum.

- RELATIVE (0): Positioned relative to the end of the chat history (e.g., "Depth 4" means 4 messages from the bottom). public/scripts/PromptManager.js37-38 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/PromptManager.js#L37-L38)

- ABSOLUTE (1): Positioned at a fixed location in the prompt list, regardless of chat length. public/scripts/PromptManager.js39-40 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/PromptManager.js#L39-L40)

## Component Assembly

### 1. System Prompts and Story Strings

The system prompt defines the base behavior of the model. Users can select from presets or define character-specific prompts.

- Default Prompt: "Write {{char}}'s next reply in a fictional chat between {{charIfNotGroup}} and {{user}}." public/scripts/openai.js101 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/openai.js#L101-L101)

- Reasoning Injection: If enabled, reasoning templates (e.g., "Think XML") guide model thought processes. The extractReasoningFromData function parses these from provider-specific fields like reasoning_content. public/scripts/reasoning.js93-148 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/reasoning.js#L93-L148)

### 2. Context Templates and Story String

The Context Template determines how character data (Description, Personality, Scenario) is structured before the chat history begins.

- Macro Substitution: Templates are processed through renderTemplateAsync to replace placeholders like {{char}} and {{user}}. public/scripts/PromptManager.js11 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/PromptManager.js#L11-L11)

- Reasoning Templates: Templates like DEFAULT_REASONING_TEMPLATE define how thought blocks are wrapped (prefix/suffix/separator). public/scripts/reasoning.js22-35 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/reasoning.js#L22-L35)

### 3. Post-Processing and Conversion

Once prompts are assembled into a message array, they may require backend conversion to fit specific provider requirements. This is handled by src/prompt-converters.js.

## Implementation Details

### Prompt Migration and Defaults

SillyTavern maintains backward compatibility for prompts through migration functions that move legacy settings into the PromptManager structure.

- Migration: registerPromptManagerMigration handles moving main_prompt, nsfw_prompt, and jailbreak_prompt from settings into the prompts array. public/scripts/PromptManager.js45-75 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/PromptManager.js#L45-L75)

### Tool Calling Integration

If the model supports tool calling, the pipeline integrates tool definitions and results.

- Tool Registration: Tools are defined with a name, description, and JSON schema for parameters. public/scripts/tool-calling.js35-44 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/tool-calling.js#L35-L44)

- In-Context Results: Tool results are injected back into the message array for follow-up generation. public/scripts/tool-calling.js16-25 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/tool-calling.js#L16-L25)

### Reasoning and Thinking Extraction

For models that provide a separate "thinking" field (like DeepSeek or Claude 3.7), the pipeline extracts this data for display and future context.

- Extraction: extractReasoningFromData maps provider-specific keys (e.g., reasoning_content for DeepSeek, thinking for Claude) to a unified format. public/scripts/reasoning.js113-127 (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/public/scripts/reasoning.js#L113-L127)
