This document describes the group chat system in SillyTavern, which enables conversations between the user and multiple AI characters simultaneously. Group chats support various activation strategies for determining which character responds next, multiple generation modes for message handling, and automated conversation flow.

For information about individual character management, see Character Management (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/Character Management) For chat storage formats, see Chat Storage and Persistence (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/Chat Storage and Persistence) For world info that applies to groups, see World Info System (https://github.com/SillyTavern/SillyTavern/blob/51ad27fb/World Info System)

## Overview and Core Concepts

The group chat system allows multiple characters to participate in a single conversation. Each group maintains its own set of members, activation strategy, generation settings, and chat history. The system determines which character speaks next based on the configured activation strategy and manages the conversation flow accordingly.

### Group Data Structure

A group is represented by a Group object with the following structure:

## Character Card Merging

In APPEND and APPEND_DISABLED modes, the system combines character cards from all group members into a single aggregated card. This allows the AI to consider all characters' descriptions, personalities, and scenarios when generating responses.

### Merging Process

The getGroupCharacterCardsLazy() function implements lazy evaluation of merged character fields.

## Bookmarks and Branching

Group chats support Checkpoints (Bookmarks) and Branching, allowing users to save specific states of a conversation and return to them or create alternative timelines.

### Branching Logic

When a branch is created from a specific message ID, the system creates a new chat file containing only messages up to that point.
