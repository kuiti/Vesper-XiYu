# DeepSeek V4 Pro / Flash returns reasoning_content must be passed back error #2754

### Description

## Describe the bug

When using DeepSeek V4 Pro (or DeepSeek V4 Flash) with Crush in thinking mode, the second turn in a conversation always fails with a 400 Bad Request error:

ERROR  Bad Request
Error from provider (DeepSeek): The `reasoning_content` in the thinking mode must be passed back to the API.
The first message completes successfully, but any follow-up message triggers this error immediately.

## To Reproduce

Steps to reproduce the behavior:

- Configure Crush to use DeepSeek with a model that supports thinking mode (e.g., deepseek-v4-pro, deepseek-v4-flash).
Example crush.json snippet:

{
  "providers": {
    "deepseek": {
      "type": "openai-compat",
      "base_url": "https://opencode.ai/zen/go/v1",
      "api_key": "API_KEY",
      "models": [
        {
          "id": "deepseek-v4-pro",
          "name": "DeepSeek V4 Pro"
        }
      ]
    }
  }
}

- 
Start a conversation and send several messages (usually 3–6 turns); the early turns complete without error.

- 
Continue the conversation normally. At some point (not always the second turn), Crush returns the 400 error above.

- 
Send any further message in the same session; every request now fails with the same error.

## Expected behavior

The second turn (and subsequent turns) should succeed, with Crush correctly handling the reasoning_content field returned by DeepSeek’s thinking mode.

## Additional context

This is a known requirement from the DeepSeek API: when thinking mode is enabled, every assistant message in the conversation history must include the reasoning_content field (the model’s internal thoughts) returned in the previous response. Failing to pass it back causes a 400 error.

## Suggested fix

Store the reasoning_content from each assistant response and include it when constructing the next request’s messages (the assistant message should contain both content and reasoning_content).
