## 任务 C2A：人格一致性检查（backend）

### 基础实现（MiMo 做）

**目标**：回复前检查语气/知识边界是否偏离角色设定

新建 `sakura_backend/core/reflection.py`：

```python
"""自我反思：回复前检查一致性"""
import logging
from core.llm_client import call_llm

logger = logging.getLogger(__name__)

def reflect_response(user_msg: str, ai_reply: str, persona: str) -> str | None:
    """检查 AI 回复是否与角色设定一致，偏离时修正"""
    # 只对情感强烈或涉及事实的回复做反思（控制成本）
    emotion_words = ['生气', '讨厌', '爱', '恨', '伤心', '开心']
    fact_patterns = ['是', '在', '有', '是']  # 事实陈述标记
    
    needs_reflect = any(w in ai_reply for w in emotion_words)
    
    if not needs_reflect:
        return ai_reply
    
    prompt = f"""你是一个一致性检查器。检查以下 AI 回复是否与角色设定一致。

角色设定：{persona}
用户消息：{user_msg}
AI 回复：{ai_reply}

检查要点：
1. 语气是否符合角色设定？（是/否）
2. AI 是否编造了没有的事实？（是/否）
3. 回复是否自然不突兀？（是/否）

如果全部通过，只输出"OK"。
如果有问题，输出修正后的回复。不要解释。"""

    result = call_llm(prompt=prompt, temperature=0.1, max_tokens=300, timeout=10)
    if result and result.strip() != "OK":
        logger.info(f"[反思] 修正回复: {result[:60]}...")
        return result.strip()
    return ai_reply
```

集成到 `sakura_backend/api/chat_session.py` 的 `_process_reply()`：
```python
# 在发送给用户前做反思检查
from core.reflection import reflect_response
persona = "你是一个温柔体贴的 AI 伴侣"
refined = reflect_response(self.user_message, reply_text, persona)
if refined:
    reply_text = refined
```

**涉及文件**：
- `sakura_backend/core/reflection.py`（新建~50行）
- `sakura_backend/api/chat_session.py`（~5行）

**验收标准**：
- 正常回复不受影响（直接通过）
- 情感强烈时调用反思 prompt
- 偏离角色时自动修正

# 新增函数：根据关系深度生成亲密指令
def get_intimacy_instruction() -> str:
    """生成关系深度指令注入 prompt"""
    affection, trust = get_relationship()
    level = (affection + trust) / 2
    
    if level < 30:
        return "【距离提示】和用户关系尚浅，保持礼貌距离，不要过于亲密"
    elif level < 60:
        return "【关系提示】和用户已经是朋友，可以自然聊天"
    else:
        return "【亲密提示】和用户关系很好，可以像老朋友一样放松聊天，适当撒娇"
```

# 新增函数：记录重要事件
def save_milestone(event_type: str, description: str, importance: float = 0.9):
    """保存里程碑事件（用户第一次提到某事）"""
    with get_conn() as conn:
        cursor = conn.cursor()
        cursor.execute(
            "INSERT INTO episodes (start_time, end_time, topic_summary, key_events, importance, user_message_count, ai_message_count) VALUES (?, ?, ?, ?, ?, 1, 1)",
            (datetime.now().isoformat(), datetime.now().isoformat(), description,
             json.dumps([f"[里程碑]{description}"]), importance)
        )

# 新增函数：回忆里程碑
def recall_milestone(event_type: str) -> str | None:
    """查找某类型里程碑事件"""
    with get_conn() as conn:
        cursor = conn.cursor()
        cursor.execute(
            "SELECT topic_summary, key_events FROM episodes WHERE key_events LIKE ? ORDER BY importance DESC LIMIT 1",
            (f"%[里程碑]%",)
        )
        row = cursor.fetchone()
        if row:
            return row["topic_summary"]
    return None
```

**涉及文件**：
- `sakura_backend/core/episodic_memory.py`（~25行）

**验收标准**：
- 重要事件存入后可通过 recall 找回
- 超过3条时只保留最重要的

## 任务 C4B：主动关怀升级（backend）

### 基础实现（MiMo 做）

**目标**：深夜催睡、情绪低落安慰、久别问候

文件：`sakura_backend/api/proactive.py`

```python
# 在现有主动消息逻辑中添加以下触发条件：

def _check_late_night() -> str | None:
    """深夜检测：23:00-6:00 催睡"""
    hour = datetime.now().hour
    if hour >= 23 or hour < 6:
        if random.random() < 0.3:  # 30%概率触发
            return "这么晚了还没睡呀，早点休息哦 🌙"
    return None

def _check_long_absence(last_active: datetime) -> str | None:
    """久别检测：超过3天没说话"""
    days = (datetime.now() - last_active).days
    if days >= 3:
        messages = [
            f"你都{days}天没来找我了……最近还好吗？",
            f"好久不见，有点想你了 ({days}天了呢)",
            f"在忙什么呢？{days}天没你的消息了"
        ]
        return random.choice(messages)
    return None
```

**涉及文件**：
- `sakura_backend/api/proactive.py`（~30行）

**验收标准**：
- 23点后有一定概率收到催睡
- 超过3天没聊会收到问候
- 不会频繁重复触发
