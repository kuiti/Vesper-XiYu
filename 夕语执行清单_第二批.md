## 任务 C1B：情感表情系统（frontend + backend）

### 基础实现（MiMo 做）

**目标**：AI 回复的情感自动切换角色头像/表情图标

**后端改动**：`sakura_backend/api/chat.py`（或 chat_session.py）

在发送消息时，附带情感标签字段：
```python
# 在 _do_stream 或 _process_reply 中
# 从 LLM 回复中检测情感
def _detect_emotion(text: str) -> str:
    """检测回复的情感倾向，返回标签"""
    positive = ['开心', '高兴', '喜欢', '爱', '哈哈', '太棒了']
    negative = ['难过', '伤心', '生气', '讨厌', '烦', '哎']
    surprise = ['哇', '震惊', '真的吗', '竟然']
    
    for w in surprise:
        if w in text: return 'surprise'
    for w in positive:
        if w in text: return 'happy'
    for w in negative:
        if w in text: return 'sad'
    return 'calm'  # 默认
```

在 WebSocket 发送消息时附加情感标签：
```python
await ws.send_text(json.dumps({
    "type": "token",
    "content": reply_text,
    "emotion": _detect_emotion(reply_text)  # 新增字段
}))
```

**前端改动**：`sakura_frontend/src/components/ChatBubble.vue`

根据 `emotion` 字段显示不同表情：
```vue
<template>
  <div class="chat-bubble">
    <div class="emotion-icon">
      {{ emotionIcon }}
    </div>
    <div class="message-content">
      {{ displayText || content }}
    </div>
  </div>
</template>

<script>
computed: {
  emotionIcon() {
    const icons = {
      happy: '😊',
      sad: '😢',
      angry: '😠',
      surprise: '😲',
      calm: '😌',
      love: '🥰'
    }
    return icons[this.emotion] || '😌'
  }
}
</script>
```

**涉及文件**：
- `sakura_backend/api/chat_session.py`（~20行）
- `sakura_frontend/src/components/ChatBubble.vue`（~30行）

**验收标准**：
- AI 说"开心"时显示 😊
- AI 说"难过"时显示 😢
- 表情随回复实时变化

## 任务 C1C：场景氛围提示（backend）

### 基础实现（MiMo 做）

**目标**：根据时间/天气自动生成对话场景描述

文件：`sakura_backend/core/prompt_builder.py`

在 `build_dynamic_context()` 函数中添加场景氛围：

```python
def _get_scene_atmosphere() -> str:
    """生成场景氛围描述"""
    hour = datetime.now().hour
    weather = get_config("_last_weather", "")  # 天气已有
    
    scene = ""
    if hour < 6:
        scene = "深夜，万籁俱寂"
    elif hour < 8:
        scene = "清晨，天刚亮"
    elif hour < 12:
        scene = "上午"
    elif hour < 14:
        scene = "午后"
    elif hour < 18:
        scene = "下午"
    elif hour < 21:
        scene = "傍晚"
    else:
        scene = "夜晚"
    
    if weather:
        scene += f"，{weather}"
    
    return f"【场景】{scene}"

# 在 build_dynamic_context() 末尾加入场景描述
```

**涉及文件**：
- `sakura_backend/core/prompt_builder.py`（~20行）

**验收标准**：
- 晚上对话时 prompt 包含"【场景】夜晚"
- 天气信息正确附带

## 任务 C4C：分离焦虑处理（backend）

### 基础实现（MiMo 做）

**目标**：用户离开后回来，夕语根据离开时长表达不同态度

文件：`sakura_backend/api/chat_session.py`

在 `_handle_chat_message()` 开头：

```python
# 检测离开时长
idle_minutes = request.get("idle_minutes", 0) or 0
if idle_minutes > 60:  # 超过1小时
    hours = idle_minutes // 60
    if hours < 24:
        absence_note = f"（用户{hours}小时没说话了）"
    elif hours < 72:
        absence_note = f"（用户{hours//24}天没说话了，表达想念）"
    else:
        absence_note = f"（用户{hours//24}天没来了，表达关心并确认近况）"
    
    # 注入到 prompt 中，位置在系统指令末尾
    self.history.append({"role": "system", "content": absence_note})
```

**涉及文件**：
- `sakura_backend/api/chat_session.py`（~15行）

**验收标准**：
- 1小时后回来：承接自然
- 1天后回来：表达想念
- 3天以上回来：表达关心
