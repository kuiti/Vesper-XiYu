## 任务 C6A：结构化记忆表格（backend + frontend）

### 基础实现（MiMo 做）

**目标**：借鉴 st-memory-enhancement，提取对话为结构化表格，每次都注入 prompt

新建 `sakura_backend/core/memory_table.py`：

```python
"""结构化记忆表格：自动填充、用户可编辑、每次都注入"""
import json
from core.db import get_conn

# 默认表格模板
DEFAULT_TEMPLATE = {
    "基本信息": ["姓名", "年龄", "城市", "职业"],
    "偏好": ["喜欢的食物", "爱好", "喜欢的音乐"],
    "当前状态": ["正在忙什么", "最近心情", "目标"],
}

def get_table_content() -> list:
    """获取表格内容（用于注入 prompt）"""
    rows = []
    with get_conn() as conn:
        cursor = conn.cursor()
        cursor.execute("SELECT key, value FROM user_profile WHERE confidence >= 0.6 AND key NOT LIKE '_fact_%' AND key NOT LIKE '_%' LIMIT 15")
        for r in cursor.fetchall():
            rows.append(f"{r['key']}: {r['value']}")
    return rows

def get_table_prompt() -> str:
    """生成表格格式的 prompt 注入文本"""
    rows = get_table_content()
    if not rows:
        return ""
    table = "【关于你的信息】\n"
    for row in rows:
        table += f"  {row}\n"
    return table
```

集成到 pipeline（`core/prompt_pipeline.py`）：
```python
class MemoryTablePipe(PromptPipe):
    """结构化记忆表格注入（每次对话都注入）"""
    zone = "dynamic"
    def process(self, ctx):

            return None
        from core.memory_table import get_table_prompt
        return get_table_prompt()
```

## 任务 C6B：双通道记忆验证（backend）

### 基础实现（MiMo 做）

文件：`sakura_backend/core/vector_store/search.py`

在 `search_similar()` 的向量检索后、注入 prompt 前，加一层过滤：

```python
def _rerank_by_entity_overlap(query: str, candidates: list, top_k: int) -> list:
    """实体重叠度过滤：丢弃与查询无关的候选"""
    from core.profile_builder import extract_entities_from_text
    query_entities = set(e[0] for e in extract_entities_from_text(query))
    if not query_entities:
        return candidates[:top_k]  # 无实体时不过滤
    
    scored = []
    for doc, dist, meta, hybrid, emb in candidates:
        doc_entities = set(e[0] for e in extract_entities_from_text(doc))
        overlap = len(query_entities & doc_entities)
        if overlap >= 1:  # 至少一个实体重叠才保留
            scored.append((doc, dist, meta, hybrid, emb))
        else:
            scored.append((doc, dist + 0.3, meta, hybrid * 0.5, emb))  # 降权但不丢弃
    
    scored.sort(key=lambda x: x[1])  # 按 dist 排序
    return scored[:top_k]
```

## 任务 C6C：记忆图谱可视化（frontend）

### 基础实现（MiMo 做）

**目标**：前端展示力导向关系图

由于不需要改后端（已有 `entities` 和 `knowledge_graph` 表），纯前端组件：

新建 `sakura_frontend/src/components/MemoryGraph.vue`：

```vue
<template>
  <div class="memory-graph" ref="graphContainer">
    <canvas ref="canvas" @mousemove="onMouseMove"></canvas>
  </div>
</template>

<script>
export default {
  data() {
    return { nodes: [], edges: [], ctx: null }
  },
  async mounted() {
    // 1. 从 API 获取实体数据
    const resp = await fetch('/api/entities')
    const data = await resp.json()
    // data = [{entity_text, entity_type, linked_memories}, ...]
    
    // 2. 布局：简单力导向
    this.nodes = data.map((e, i) => ({
      id: i, label: e.entity_text, type: e.entity_type,
      x: Math.random() * 500, y: Math.random() * 500,
      vx: 0, vy: 0
    }))
    
    // 3. 画布渲染循环
    this.ctx = this.$refs.canvas.getContext('2d')
    this.animate()
  },
  methods: {
    animate() {
      // 简单的力导向模拟
      this.nodes.forEach(n => {
        // 排斥力
        this.nodes.forEach(other => {
          if (n.id !== other.id) {
            const dx = n.x - other.x, dy = n.y - other.y
            const dist = Math.sqrt(dx*dx + dy*dy) || 1
            n.vx += dx / dist * 0.1
            n.vy += dy / dist * 0.1
          }
        })
        // 速度衰减
        n.vx *= 0.9; n.vy *= 0.9
        n.x += n.vx; n.y += n.vy
      })
      
      // 绘制
      this.ctx.clearRect(0, 0, 500, 500)
      this.nodes.forEach(n => {
        this.ctx.beginPath()
        this.ctx.arc(n.x, n.y, 20, 0, Math.PI * 2)
        this.ctx.fillStyle = n.type === 'PERSON' ? '#4CAF50' : '#2196F3'
        this.ctx.fill()
        this.ctx.fillStyle = 'white'
        this.ctx.fillText(n.label, n.x - 10, n.y + 5)
      })
      
      requestAnimationFrame(() => this.animate())
    }
  }
}
</script>
```

**涉及文件**：
- `sakura_frontend/src/components/MemoryGraph.vue`（新建~80行）

**验收标准**：
- 页面显示力导向关系图
- 节点上显示实体名称
- 不同类型的节点颜色不同

```python
# 在 generate_summary_for_tier() 中修改：
# 当 level == 3 时，保留前两级摘要完整内容

def get_stack_summary(level: int) -> str:
    """获取指定级别以上的摘要堆叠"""
    with get_conn() as conn:
        cursor = conn.cursor()
        cursor.execute(
            "SELECT level, summary FROM tiered_summary WHERE level <= ? AND remaining_days > 0 ORDER BY level ASC, created_at DESC",
            (level,)
        )
        rows = cursor.fetchall()
    
    parts = []
    seen = set()
    for r in rows:
        if r["summary"] not in seen:
            seen.add(r["summary"])
            parts.append(f"[L{r['level']}] {r['summary']}")
    return "\n".join(parts[:10])  # 最多保留10条
```
