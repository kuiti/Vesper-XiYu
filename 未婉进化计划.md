# 未婉进化计划
## JARVIS 级 AI 管家 — 独立智能体，而非应用附属

> 对标参考：JARVIS（Iron Man — 自主智能管家）、Cortana（微软 — 系统级助手）
> 代码关系：与夕语共享核心层（记忆/LLM/DB），但 api/ 和 frontend/ 半独立
> 定位：未婉是一个**有自主能力的管家**，不是"夕语的工具模式"
> 用户画像：需要 AI 管理日程/环境/任务的深度用户
> 最后更新：2026-06-22

---

## 总体路线图

```
未婉（JARVIS 级 AI 管家）
├── Phase W1: 感知层（听见/看见/读到）
│   语音唤醒 + 系统监听 + 信息采集
│       ↓
├── Phase W2: 认知层（理解/推理/决策）
│   结论引擎 + 优先级判断 + 自主规划
│       ↓
├── Phase W3: 执行层（动手做）
│   工具链 + 系统控制 + 环境集成
│       ↓
├── Phase W4: 交互层（怎么和你沟通）
│   语音对话 + 通知/报告 + 可视化仪表盘
│       ↓
└── Phase W5: 自主层（不需要指令也能做事）
    监控 + 预警 + 自动化工作流
```

## 未婉与夕语的关系

```
夕语是"陪你聊天的人" → 情感连接、陪伴感、有趣
未婉是"帮你做事的人" → 自主执行、系统监控、任务管理

共享核心：同一套记忆系统 + 同一个 LLM + 同一份用户画像

不共享：
- Prompt 策略（夕语→感性陪伴；未婉→理性执行）
- 行为规则（夕语→自然随性；未婉→准确高效）
- 前端布局（夕语→对话优先；未婉→仪表盘优先）
- API 端点（夕语→chat/emotion；未婉→task/system）

部署关系：可以独立运行未婉（无夕语），也可以同时运行两者互通数据
```

## 实施总表

| # | 方向 | 具体实施方案 | 改动文件 | 工期 |
|---|------|-------------|---------|------|
| **Phase W1: 感知层 — 未婉能"听/看/读"** |
| W1A | 语音唤醒 | 持续监听唤醒词（如"嘿未婉"）；唤醒后开始录音→ASR转文字→执行→TTS回复；类似 Alexa 的唤醒-命令-反馈循环 | `core/wake_engine.py`(新) `gateway/voice.py`(新) | 6天 |
| W1B | 系统状态监听 | 监听系统资源（CPU/内存/磁盘/网络）；异常时主动报告；类似 JARVIS 的"Sir，网络延迟偏高" | `core/system_monitor.py`(新) | 4天 |
| W1C | 信息采集引擎 | 定时抓取用户关注的 RSS/天气/新闻/股价；采集结果入库供查询和注入 | `core/info_collector.py`(新) | 4天 |
| W1D | 文件系统感知 | 监听指定目录的文件变化（新建/修改/删除）；自动索引新文件；支持问答"帮我找上周的PPT" | `core/file_watcher.py`(新) | 4天 |
| **Phase W2: 认知层 — 未婉能"想"** |
| W2A | 自主结论引擎 | 综合多源信息（对话+系统状态+日程+环境）生成行动建议；优先级排序：紧急>重要>日常 | `core/conclusion_engine.py`(增强) | 5天 |
| W2B | 任务规划器 | 用户给模糊目标→未婉拆解为步骤；例："准备出差"→查天气→查航班→设提醒→打包清单 | `core/task_planner.py`(新) | 5天 |
| W2C | 优先级判断 | 新事件打断时判断是否值得中断当前任务；紧急（系统崩溃/重要会议）→立即通知；非紧急→汇总后报告 | `core/priority_engine.py`(新) | 4天 |
| **Phase W3: 执行层 — 未婉能"做"** |
| W3A | 工具链引擎 | 多步工具调用编排：规划→执行→检查→修正；工具结果自动传递 | `core/tool_planner.py`(新) | 5天 |
| W3B | 系统控制 | 执行系统命令（需授权）；控制应用（打开/关闭/切换）；文件操作（复制/移动/删除） | `core/system_control.py`(新) | 5天 |
| W3C | 环境集成 | 智能家居（灯光/空调/窗帘通过 HTTP/MQTT）；日历（读写 Google Calendar/Outlook）；邮件（读取/回复/摘要） | `gateway/smarthome.py`(新) `gateway/calendar.py`(新) | 7天 |
| W3D | 网络请求代理 | 自动完成需要认证的 HTTP 请求；Cookie/Token 管理；API 编排 | `core/http_agent.py`(新) | 4天 |
| **Phase W4: 交互层 — 怎么和未婉沟通** |
| W4A | 语音对话 | 全双工语音：TTS 播报 + STT 识别 + 自然打断；类似 JARVIS 的自然语音交互 | `gateway/voice.py` | 6天 |
| W4B | 通知系统 | 分级通知：紧急→弹窗+语音；重要→桌面通知；普通→汇总条；整合系统 toast + 移动推送 | `core/notify.py`(新) `api/` | 4天 |
| W4C | 仪表盘 UI | 新前端页面：系统状态卡片、任务队列、日程视图、环境控制面板；非聊天式，而是信息面板式 | `frontend/dashboard/`(新) | 6天 |
| W4D | 定期报告 | 每日早报（天气+日程+重要事件）；每周总结（完成任务+系统状态+建议）；自动推送到用户 | `core/report_engine.py`(新) | 4天 |
| **Phase W5: 自主层 — 不需要指令也能做事** |
| W5A | 自动化工作流 | 用户设定触发条件→自动执行动作；例："每天 7 点查天气，如果下雨提醒我带伞"；IFTTT 风格 | `core/workflow_engine.py`(新) | 6天 |
| W5B | 预警系统 | 设置监控指标+阈值；异常时自动预警（磁盘>90%/CPU温度过高/长时间未备份）；智能分级：信息/警告/紧急 | `core/alert_engine.py`(新) | 5天 |
| W5C | 习惯学习 | 分析用户行为模式→自动优化；例：用户连续 3 天 11 点还在线→未婉建议"要不要设个睡眠提醒" | `core/habit_learner.py`(新) | 5天 |
| W5D | 自主汇报 | 无事时安静；有事时主动报告；JARVIS 风格："Sir，有件事你可能想知道……" | `core/conclusion_engine.py` + `notify.py` | 3天 |
